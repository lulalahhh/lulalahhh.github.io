mysql日志

2025/4/11 日志

redolog:

记录的是数据页的物理变化,服务宕机可用来同步数据,

undolog:

记录的是对数据页的逻辑变化,当事务回滚,通过逆操作恢复原来的数据

redolog保证了事务的持久性,undolog保证了事务的原子性
redo log 记录了此次事务「修改后」的数据状态 ,undo log 记录了此次事务「修改前」的数据状态
redo log 是 Innodb 存储引擎实现的日志；binlog 是 MySQL 的 Server 层实现的日志。
binlog 是追加写 redo log 是循环写
